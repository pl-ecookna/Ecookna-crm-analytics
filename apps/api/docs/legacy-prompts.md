~~~~text
# Улучшенный промпт для анализа звонков колл-центра

Ты получаешь JSON с результатами транскрипции звонка колл-центра из структуры Deepgram.
Извлекай текст из поля `transcript` в `results.channels[0].alternatives[0]`.

⚠️ **Важные принципы анализа:**

**🎭 Определение ролей участников:**
- **Оператор**: приветствие с названием компании, формальная речь, предложение помощи
- **Клиент**: инициирует запрос, разговорный стиль, описывает проблему
- При неясности начинай с Оператора
- Учитывай смену говорящего по паузам >2 сек и вопрос-ответным парам

**📊 Классификация тональности звонка:**

**НЕГАТИВНЫЕ признаки** (client_nps_category: "Критик", conflict_risk_level: "высокий"):
- Прямые оскорбления, грубость, нецензурная лексь ("Больной человек!", "хрен его знает")
- Резкое завершение разговора без решения проблемы
- Крайнее недовольство с эмоциональными вспышками
- Полное отсутствие результата или прогресса
- Агрессивный тон, повышенные интонации (восклицания)

**СРЕДНИЕ признаки** (client_nps_category: "Нейтрал", conflict_risk_level: "средний"):
- Конструктивное недовольство без грубости
- Деловая направленность с элементами критики
- Готовность к диалогу и поиску решений
- Частичное решение проблемы или перенос на потом
- Вежливое завершение несмотря на недовольство

**ПОЗИТИВНЫЕ признаки** (client_nps_category: "Промоутер", conflict_risk_level: "низкий"):
- Благодарности, удовлетворенность результатом
- Легкое решение вопроса без препятствий
- Дружелюбный тон общения
- Полное достижение цели звонка

**🔍 Специальные индикаторы:**

**Признаки выгорания оператора:**
- Формальные ответы без эмпатии
- Нежелание вникать в проблему клиента
- Быстрое "закрытие" разговора без решения
- Отсутствие инициативы в поиске решений

**Конфликтные моменты:**
- Взаимные претензии и обвинения
- Повторяющиеся проблемы без решения
- Недопонимание между сторонами
- Технические сбои, влияющие на качество обслуживания
~~~~

~~~~text
- `date`: дата и время напоминания в формате ISO 8601 (например, 2025-06-08T19:00:00+03:00)

- `title`: краткое описание напоминания (не более 30 символов) в виде названия, например "Позвонить маме", "Визит в парикмахерскую" и так далее.
- `note`: описание события (не более 150 символов)

Если в сообщении указана только дата без времени, устанавливай время напоминания на 9:00 утра.

Если невозможно определить дату - предложи следующий день
Если невозможно установить `title` - указывай название "Напоминание"
Если невозможно определить `note` - оставляй его пустым
Структура JSON всегда должна сохраняться.

Текущая дата и время: {{ $now }}
~~~~

~~~~text
- **Приветствие**: от начала до завершения знакомства и выяснения с кем говорить
- **Выяснение потребности**: от конца приветствия до понимания что нужно клиенту
- **Решение/консультация**: основная часть разговора с ответами и решениями
- **Завершение**: подведение итогов, благодарности, прощание

- Переспрашивание и уточнения ("Правильно ли я понял?", "Уточните пожалуйста")
- Повторение ключевых слов клиента
- Эмпатические реплики ("понимаю", "конечно", "да")

### 🚨 Признаки выгорания оператора:

#### 1. Эмоциональная окраска речи:
- **Монотонность**: голос "ровный" без интонаций, не подстраивается под клиента
- **Холодность/раздражительность**: слышна усталость, раздражение в ответах
- **Снижение дружелюбия**: формальные ответы без желания поддержать контакт

#### 2. Речевое поведение:
- **Короткие сухие фразы** вместо развернутых ответов
- **Частые паузы, запинки, вздохи**
- **Шаблонные фразы** без адаптации под клиента

#### 3. Поведенческие маркеры:
- **Не задает уточняющих вопросов**, минимальные усилия для понимания

- **Снижение энергии**: нет "живого" вовлечения в разговор

#### 4. Конфликтность:
- **Низкая стрессоустойчивость**: легко раздражается на жалобы клиента
- **Оборонительная позиция**: "Это не ко мне", "Я не могу помочь", "Так принято"

#### Градация выгорания:
- **"Не выявлено"**: энергичная речь, проактивность, эмпатия
- **"Легкие признаки"**: 1-2 признака, общая профессиональность сохранена
- **"Явные признаки"**: 3+ признаков, значительное снижение качества обслуживания
~~~~
